# UI/UX改善チェックリスト

## 優先度: 高（使えない問題）

### ボタンとインタラクション
- [ ] すべてのボタンにタップフィードバック（スケール + 不透明度）を追加
- [ ] Pressableの`hitSlop`を設定してタップ領域を拡大（最低44x44px）
- [ ] ローディング中はボタンを無効化し、インジケーターを表示
- [ ] 連続タップを防ぐデバウンス処理を追加

### フォーム入力
- [ ] キーボードが入力欄を隠さないようにKeyboardAvoidingViewを最適化
- [ ] フォーカス時に入力欄を自動スクロール
- [ ] エラーメッセージを入力欄の直下に表示
- [ ] 必須項目に`*`マークを追加
- [ ] プレースホルダーテキストを具体的に（例: "例: 2026-03-15"）

### レイアウトとスクロール
- [ ] SafeAreaを正しく適用（タブバーに隠れない）
- [ ] スクロール可能な領域を明確に（スクロールインジケーター）
- [ ] 長いテキストの省略表示（numberOfLines + ellipsizeMode）
- [ ] 画像の読み込み失敗時のフォールバック表示

### エラーハンドリング
- [ ] ネットワークエラー時の再試行ボタンを追加
- [ ] エラーメッセージを日本語でわかりやすく
- [ ] 空の状態（Empty State）に適切なメッセージとアクションを表示
- [ ] ローディング状態を明確に表示（スケルトンスクリーン）

## 優先度: 中（使いにくい問題）

### ナビゲーション
- [ ] 戻るボタンの位置を統一（左上）
- [ ] モーダルの閉じるボタンを明確に
- [ ] タブバーのアイコンとラベルを一致させる
- [ ] 深い階層からのホームへの戻り方を提供

### フィードバック
- [ ] 成功時のトースト通知を追加
- [ ] 削除などの破壊的操作に確認ダイアログを表示
- [ ] 進行状況をプログレスバーで表示
- [ ] ハプティックフィードバックを追加（重要なアクション）

### テキストとタイポグラフィ
- [ ] 行間を適切に設定（lineHeight: fontSize * 1.5）
- [ ] コントラスト比を確保（WCAG AA基準）
- [ ] フォントサイズを統一（14px, 16px, 18px, 24px, 32px）
- [ ] 長いテキストの折り返しを適切に

### パフォーマンス
- [ ] 画像の遅延読み込みを実装
- [ ] FlatListにgetItemLayoutを追加
- [ ] 重い計算をuseMemoでメモ化
- [ ] 不要な再レンダリングを防ぐ（React.memo）

## 優先度: 低（改善の余地）

### アクセシビリティ
- [ ] accessibilityLabelを追加
- [ ] accessibilityHintを追加
- [ ] accessibilityRoleを設定
- [ ] スクリーンリーダー対応

### アニメーション
- [ ] ページ遷移にトランジションを追加
- [ ] リスト項目の表示にフェードインを追加
- [ ] モーダルの開閉にスライドアニメーションを追加
- [ ] ボタンのホバー効果（Web）

### デザインの一貫性
- [ ] カラーパレットを統一
- [ ] ボタンのスタイルを統一（Primary, Secondary, Tertiary）
- [ ] カードのスタイルを統一（影、角丸、パディング）
- [ ] アイコンのサイズを統一（16px, 20px, 24px, 32px）
