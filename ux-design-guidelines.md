# UXデザインガイドライン

Kimito-Link-Voiceを参考に、ユーザビリティとユーザー体験を重視したデザイン指針をまとめる。

---

## 1. オンボーディング体験

### 3ステップ説明の原則

初めてのユーザーがサービスの使い方を即座に理解できるよう、**3ステップ以内**で説明する。各ステップはアイコン＋短いテキスト（10文字以内）で構成し、視覚的に流れが分かるようにする。

| ステップ | 内容 | アイコン例 |
|---------|------|-----------|
| 1 | ログイン | 鍵・ログインアイコン |
| 2 | 選択・参加 | タップ・チェックアイコン |
| 3 | 完了・達成 | 星・完了アイコン |

---

## 2. ナビゲーション設計

### 迷わない導線

ユーザーが「次に何をすればいいか」を常に明確にする。主要なCTA（Call To Action）ボタンは画面内に1つだけ配置し、色とサイズで目立たせる。

**CTAボタンの優先度:**
- **プライマリ**: 背景色付き、大きめ（例: Twitterでログイン）
- **セカンダリ**: 枠線のみ、中サイズ（例: 詳細を見る）
- **ターシャリ**: テキストリンク、小さめ（例: スキップ）

---

## 3. 視覚的階層

### 情報の優先順位を明確に

重要な情報ほど大きく、目立つ位置に配置する。ユーザーの視線は「Z型」または「F型」に動くことを意識し、左上から右下への流れを設計する。

| 要素 | フォントサイズ | 配置 |
|-----|--------------|------|
| ヒーローテキスト | 28-36px | 画面上部中央 |
| セクション見出し | 20-24px | 各セクション先頭 |
| カードタイトル | 16-18px | カード内上部 |
| 説明テキスト | 14-16px | タイトル下 |
| 補足・注釈 | 12-14px | 下部または薄い色 |

---

## 4. カードデザイン

### 一貫性のあるカードUI

コンテンツはカード形式で表示し、以下の要素を統一する。

- **角丸**: 12-16px（柔らかい印象）
- **影**: 軽いドロップシャドウ（0 2px 8px rgba(0,0,0,0.1)）
- **余白**: 内側16px、カード間12-16px
- **背景**: 白または薄いグレー（#F8F9FA）

### カード内の構成

1. **ビジュアル**（画像・アイコン）: 上部または左側
2. **タイトル**: 太字、1-2行以内
3. **説明**: 通常ウェイト、2-3行以内
4. **アクション**: ボタンまたはリンク、右下または下部

---

## 5. レスポンシブデザイン

### 8段階ブレークポイント

すべての機能・コンポーネントは最初からレスポンシブ対応必須。モバイルファースト設計で、280px（折りたたみスマホ）〜4K（3840px）まで完全対応する。

| デバイス | 幅 | 対象デバイス例 |
|---------|---|--------------|
| 超小型 | ≤280px | Galaxy Fold 5（折りたたみ） |
| 極小 | 281-360px | iPhone SE, Galaxy S8+ |
| 小型 | 361-400px | iPhone 12 mini, Pixel 7 |
| 標準スマホ | 401-480px | iPhone 12 Pro, iPhone 14 Pro Max |
| 大型スマホ/小型タブレット | 481-768px | iPad Mini, Galaxy S20 Ultra |
| 標準タブレット | 769-1024px | iPad Air, iPad Pro |
| Surface/小型PC | 1025-1366px | Surface Pro 7, Surface Duo |
| PC/大型ディスプレイ | ≥1367px | デスクトップ、4K |

### PC/スマホでの表示差異

| 要素 | スマホ | PC |
|-----|-------|-----|
| カード配置 | 縦1-2列 | 横2-3列 |
| ナビゲーション | 下部タブバー | 上部ヘッダー or サイドバー |
| ヒーロー画像 | 縦長、画面幅100% | 横長、最大幅1200px |
| フォントサイズ | 基準サイズ | 1.1-1.2倍 |
| コンテンツ最大幅 | 100% | 1200px（中央揃え） |

## 6. タッチ対応

### 最小タップエリア

iOS Human Interface Guidelinesに準拠し、タップ可能な要素は最小44x44pxを確保する。

```
.touch-target {
    min-width: 44px;
    min-height: 44px;
    padding: 12px;
}
```

### タップフィードバック

| 操作 | フィードバック |
|-----|--------------|
| ボタンタップ | scale(0.97) + 軽いハプティクス |
| リストアイテム | opacity: 0.7 |
| アイコン | opacity: 0.6 |

## 7. 色コントラストガイドライン

### 最小コントラスト比（WCAG 2.1準拠）

- **通常テキスト（18px未満）**: コントラスト比 **4.5:1** 以上
- **大きいテキスト（18px以上 または 14px太字以上）**: コントラスト比 **3:1** 以上
- **UI要素（ボタン、アイコン）**: コントラスト比 **3:1** 以上

### 推奨テキスト色

#### 白い背景（#FFFFFF）の場合

| 用途 | 色 | コントラスト比 |
|-----|-----|---------------|
| メインテキスト | #1a1a1a | 16.7:1 ✅ |
| セカンダリテキスト | #333333 | 12.6:1 ✅ |
| サブテキスト | #555555 | 8.0:1 ✅ |

#### 避けるべき組み合わせ

- 白い背景に #999999（コントラスト比 2.8:1 ❌）
- 白い背景に #cccccc（コントラスト比 1.6:1 ❌）
- #666は危険：白い背景では使えるが、他の背景では見えなくなる可能性が高い

### 覚えておくべきポイント

1. **#666は危険**: 白い背景では使えるが、他の背景では見えなくなる可能性が高い
2. **#333が安全**: ほとんどの白い背景で問題なく見える
3. **#1a1a1aが最適**: 最も濃い黒で、あらゆる場面で視認性が高い
4. **太字を活用**: font-weight: 600以上で視認性が大幅に向上
5. **テキストシャドウ**: グラデーション背景では text-shadow を使用

## 8. ブランドカラー

### 色の使い分け

| 用途 | 色 | 使用例 |
|-----|-----|-------|
| プライマリアクション | オレンジ #DD6500 | CTAボタン、重要リンク |
| ブランド・信頼 | ブルー #00427B | ヘッダー、ロゴ周り |
| 成功・完了 | グリーン #22C55E | 達成バッジ、成功メッセージ |
| 警告・注意 | イエロー #F59E0B | 期限間近、注意喚起 |
| エラー・危険 | レッド #EF4444 | エラーメッセージ、削除 |
| 背景・ニュートラル | グレー系 | カード背景、区切り線 |

## 9. クロスプラットフォーム対応

### iOS対応

- タップハイライトの除去: `-webkit-tap-highlight-color: transparent`
- スムーズスクロール: `-webkit-overflow-scrolling: touch`
- フォントサイズの自動調整防止: input, textareaは16px以上（ズーム防止）
- Safe Area対応: `env(safe-area-inset-top)` 等を使用

### Android対応

- Material Design準拠のタップ効果: `transform: scale(0.98)`
- 高DPI対応: 2x以上の解像度で画像を用意
- フォントレンダリング: `-webkit-font-smoothing: antialiased`

### Windows（Surface/PC）対応

- タッチとマウスの両対応: `@media (pointer: coarse)` / `@media (pointer: fine)`
- ホバー効果はマウスデバイスのみ適用

## 10. フィードバック設計

### ユーザーの操作に応答する

すべてのインタラクションに対して、視覚的・触覚的なフィードバックを提供する。

| 操作 | フィードバック |
|-----|--------------|
| ボタンタップ | 0.97倍に縮小 + 軽いハプティクス |
| ローディング | スケルトンUI or スピナー |
| 成功 | 緑色トースト + 成功ハプティクス |
| エラー | 赤色トースト + エラーハプティクス |
| スワイプ | 追従するアニメーション |

---

## 11. パフォーマンス体験

### 待たせない、イライラさせない

- **初期表示**: 1秒以内にファーストビュー
- **画像**: 遅延読み込み（Lazy Loading）
- **スケルトンUI**: データ取得中は骨組みを表示
- **オプティミスティックUI**: 操作結果を先に反映

---

## 12. 感情設計

### ポジティブな体験を演出

- **達成感**: 目標達成時にアニメーション＋お祝いメッセージ
- **進捗の可視化**: プログレスバー、パーセンテージ表示
- **キャラクターの活用**: 状況に応じた表情・セリフで親しみやすさを演出
- **マイクロインタラクション**: 小さな動きで楽しさを追加
