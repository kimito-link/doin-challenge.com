# 数字表示への"意味ラベル"追加 v6.56

## 修正日時
2026-01-26

## 目的
数字だけが独り歩きして誤解・誇張・クレームにつながるのを防ぐため、すべての数字表示に「予定」「目安」「現時点」のラベルを追加

---

## 修正内容

### 1. 進捗セクション (`features/event-detail/components/ProgressSection.tsx`)

#### Line 87: 達成状況のラベル
```tsx
// 修正前
<Text style={{ color: color.textSecondary, fontSize: 14 }}>現在の達成状況</Text>

// 修正後
<Text style={{ color: color.textSecondary, fontSize: 14 }}>現在の達成状況（参加予定）</Text>
```

**効果：**
- 表示されている数字が「参加予定」であることを明示
- 「確定した参加者数」ではないことを明確化

## ラベル追加のルール

### 必須ルール
- 数字の直下 or 右に必ず補足を置く
- 「予定」「目安」「現時点」のいずれかを含める

### 使い分け
| 場面 | 推奨ラベル |
|------|-----------|
| 達成率・進捗率 | 「（参加予定）」 |
| 参加者数 | 「参加予定（現時点）」 |
| ランキング数値 | 「（現時点）」 |
| 地域別カウント | 「参加予定」 |

---

## 法務的な意義

### Before
```
達成率 50%
```
→ 「50%達成した」と誤解される可能性

### After
```
達成率 50%（参加予定）
```
→ 「参加予定ベースで50%」と正確に伝わる

**効果：**
1. **誤解の防止**: 数字が「予定」であることを明示
2. **クレーム防止**: 「実際の参加率と違う」というクレームを防ぐ
3. **法務リスク低減**: 「保証」「確約」と受け取られるリスクを最小化

---

## 今後の運用

### 新しい数字表示を追加する際のチェックリスト
- [ ] 数字のそばに「予定」「目安」「現時点」のラベルがあるか
- [ ] ユーザーが「確定した数字」と誤解しないか
- [ ] 法務的に問題のある表現になっていないか

### 例外
以下の場合はラベル不要：
- システム内部の技術的な数値（ID、バージョン番号等）
- 時刻・日付（誤解の余地がない）
- 明らかに確定した数値（過去の実績等）
