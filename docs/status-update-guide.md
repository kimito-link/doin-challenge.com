# docs/status.mdの更新手順

## 目的

`docs/status.md`は、プロジェクトの現在の状態を一元管理するドキュメントです。このドキュメントを定期的に更新することで、AIと人間の両方が現在の状態を正確に把握でき、作業の重複を防げます。

---

## 更新タイミング

以下のいずれかの状況で、`docs/status.md`を更新してください：

1. **環境変数を追加・変更した場合**
2. **外部サービスの設定を変更した場合**
3. **データベースのスキーマを変更した場合**
4. **新しい機能を実装した場合**
5. **未解決の問題が解決した場合**
6. **新しい問題が発生した場合**
7. **チェックポイントを作成した場合**

---

## 更新手順

### ステップ1: `docs/status.md`を開く

```bash
# ファイルを開く
code /home/ubuntu/birthday-celebration/docs/status.md
```

### ステップ2: 該当セクションを更新

更新する内容に応じて、以下のセクションを編集します：

#### 1. 環境変数の設定状況

新しい環境変数を追加した場合：

```markdown
| 環境変数 | 設定状況 | 設定場所 | 備考 |
|---------|---------|---------|------|
| `NEW_ENV_VAR` | ✅ 設定済み | Vercel | 2026-01-30追加 |
```

#### 2. 外部サービスの設定状況

新しい外部サービスを追加した場合：

```markdown
### NewService

| 項目 | 設定状況 | 備考 |
|------|---------|------|
| アカウント作成 | ✅ 完了 | 2026-01-30確認済み |
| 動作確認 | ✅ 完了 | - |
```

#### 3. データベースの状態

データベースのスキーマを変更した場合：

```markdown
### newTableテーブル

| 項目 | 状態 | 備考 |
|------|------|------|
| `newColumn`カラムの存在 | ✅ 存在 | 2026-01-30追加 |
```

#### 4. 実装済み機能

新しい機能を実装した場合：

```markdown
### 新機能

| 機能 | 実装状況 | 備考 |
|------|---------|------|
| 新機能A | ✅ 実装済み | v6.170で実装 |
```

#### 5. 未解決の問題

新しい問題が発生した場合：

```markdown
### 2. 新しい問題

**状況**: 問題の説明

**調査結果**:
- 調査結果1 ✅
- 調査結果2 ⚠️

**次のアクション**:
1. アクション1
2. アクション2
```

問題が解決した場合は、該当セクションを削除します。

#### 6. 次のアクション

次のアクションを更新します：

```markdown
## 次のアクション

1. **アクション1**: 説明
2. **アクション2**: 説明
```

#### 7. 更新履歴

更新履歴を追加します：

```markdown
| 日付 | バージョン | 更新内容 |
|------|-----------|---------|
| 2026-01-30 | v6.170 | 新機能Aを実装 |
```

### ステップ3: ファイルを保存

ファイルを保存します。

### ステップ4: チェックポイントを作成

`docs/status.md`を更新した後は、チェックポイントを作成してください：

```bash
# バージョンを更新
# shared/version.tsのAPP_VERSIONを更新

# チェックポイントを作成
# webdev_save_checkpoint
```

## 更新例

### 例1: 環境変数を追加した場合

**状況**: `NEW_API_KEY`という環境変数を追加した

**更新内容**:

```markdown
### その他の環境変数

| 環境変数 | 設定状況 | 設定場所 | 備考 |
|---------|---------|---------|------|
| `DATABASE_URL` | ✅ 設定済み | Vercel | - |
| `NEW_API_KEY` | ✅ 設定済み | Vercel | 2026-01-30追加 |
```

```markdown
| 日付 | バージョン | 更新内容 |
|------|-----------|---------|
| 2026-01-30 | v6.170 | NEW_API_KEYを追加 |
```

### 例2: 未解決の問題が解決した場合

**状況**: サムネイル画像の表示問題が解決した

**更新内容**:

1. 「未解決の問題」セクションから該当項目を削除
2. 「実装済み機能」セクションに追加：

```markdown
### 画像表示

| 機能 | 実装状況 | 備考 |
|------|---------|------|
| サムネイル画像の表示 | ✅ 実装済み | v6.170で修正 |
```

3. 更新履歴を追加：

```markdown
| 日付 | バージョン | 更新内容 |
|------|-----------|---------|
| 2026-01-30 | v6.170 | サムネイル画像の表示問題を解決 |
```

### 例3: 新しい機能を実装した場合

**状況**: 通知機能を実装した

**更新内容**:

1. 「実装済み機能」セクションに追加：

```markdown
### 通知機能

| 機能 | 実装状況 | 備考 |
|------|---------|------|
| プッシュ通知 | ✅ 実装済み | v6.170で実装 |
| メール通知 | ✅ 実装済み | v6.170で実装 |
```

2. 更新履歴を追加：

```markdown
| 日付 | バージョン | 更新内容 |
|------|-----------|---------|
| 2026-01-30 | v6.170 | 通知機能を実装 |
```

---

## 注意事項

1. **更新は小まめに行う**: 作業が完了したら、すぐに`docs/status.md`を更新してください
2. **正確な情報を記載する**: 不確かな情報は「⚠️ 未確認」と明記してください
3. **日付とバージョンを記載する**: 更新履歴には必ず日付とバージョンを記載してください
4. **チェックポイントを作成する**: `docs/status.md`を更新した後は、チェックポイントを作成してください

---

## まとめ

`docs/status.md`を定期的に更新することで、以下のメリットがあります：

- **作業の重複を防ぐ**: AIが過去の作業を正確に把握できる
- **現在の状態を把握できる**: 人間が現在の状態を一目で確認できる
- **問題の早期発見**: 未解決の問題を明確に管理できる
- **チーム間の情報共有**: チームメンバーが現在の状態を共有できる
