# UI/UXの最終調整チェックリスト

## 概要

このドキュメントは、Web版リリース前に確認すべきUI/UXの最終調整項目をリスト化したものです。

### 4.1 プレスフィードバック

**優先順位: 機能 > フィードバック > ポリッシュ**

- [ ] 全てのボタンがプレスフィードバックを持つ
- [ ] 全てのリンクがプレスフィードバックを持つ
- [ ] 全てのカードがプレスフィードバックを持つ

**フィードバックの種類:**
- [ ] プライマリボタン: スケール + ハプティック
- [ ] リストアイテム/カード: 透明度
- [ ] アイコン/マイナーアクション: 透明度のみ

### 4.2 ハプティックフィードバック

**使用箇所:**
- [ ] ボタンタップ（プライマリアクション）: impactAsync(Light)
- [ ] トグル/スイッチ: impactAsync(Medium)
- [ ] 成功/完了: notificationAsync(Success)
- [ ] エラー/失敗: notificationAsync(Error)

**Platform.OS !== "web"で無効化:**
- [ ] Web版ではハプティックフィードバックが無効化されている

## まとめ

### 重要なポイント

1. **レスポンシブデザイン**: 全ブレークポイントで正しく表示される
2. **アクセシビリティ**: WCAG AA基準を満たす
3. **インタラクション**: プレスフィードバックが全ての要素にある
4. **空状態**: データが0件の場合、空状態が表示される
5. **エラー状態**: エラーメッセージが分かりやすい

### 確認方法

- **手動テスト**: 全画面を手動で確認
- **ユーザーテスト**: 実際のユーザーに使ってもらう
- **アクセシビリティチェック**: WCAG AA基準を満たしているか確認
- **実機テスト**: 実際のデバイスで動作確認
