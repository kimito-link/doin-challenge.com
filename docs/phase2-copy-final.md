1. **キャンセル画面**
2. **エラー画面**
3. **ローディング画面**
4. **ログアウト確認**

---

## 共通コンポーネント: LinkAuthResult

### 設計方針
- 成功/キャンセル/エラーを1つのコンポーネントで出し分け
- `type: 'success' | 'cancel' | 'error'` で切り替え
- りんくキャラクターの吹き出しで説明

うまくいかなかったみたい...

## 文言の統一ルール

### ✅ 使う言葉
- ログイン / ログアウト
- X（Twitter の代わり）
- マイページ
- 参加履歴
- お気に入り
- メッセージ管理

### ❌ 使わない言葉
- 認証
- OAuth
- API
- トークン
- セッション
- その他の専門用語

### りんく吹き出しのトーン
- 親しみやすい
- 丁寧すぎない（「です・ます」ではなく「だよ・だね」）
- 絵文字は使わない
- ポジティブな表現

---

## 実装時の注意事項

1. **文言は1文字も変えない**
   - このドキュメントの文言をそのまま使用
   - 勝手に言い回しを変えない

2. **改行位置も固定**
   - 表示領域に合わせて調整が必要な場合のみ変更

3. **ボタンテキストは短く**
   - 最大15文字以内
   - アクションが明確にわかる言葉

4. **エラーメッセージは優しく**
   - ユーザーを責めない
   - 解決策を示す
   - 技術的な詳細は隠す

5. **開発環境と本番環境の切り替え**
   - `process.env.NODE_ENV === 'development'` で判定
   - requestId は開発環境のみ表示
