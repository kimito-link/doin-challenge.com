# 動員チャレンジ 言語ポリシー（Language Policy）

**Version:** 1.0  
**Last Updated:** 2026-01-26  
**Purpose:** プロダクトの言語使用における判断基準を明文化し、法務リスクを最小化しながら、ユーザーに価値を正確に伝える

---

## 基本思想

### プロダクトの価値定義
- **提供するもの**: 参加予定の可視化、判断材料、応援の整理
- **提供しないもの**: 集客保証、成功保証、確実な結果

### メッセージングの原則
1. **断定しない**（成功・集客を言わない）
2. **意思表明・予定・目安の言葉を使う**
3. **価値は「判断材料」「可視化」「整理」で語る**
4. **煽らない**（不安を過度に刺激しない）

## 禁止ワード集（NG表現）

### カテゴリー1: 結果の保証・断定
| NG表現 | 理由 | OK表現 |
|--------|------|--------|
| 売れる | 結果を保証 | 判断材料になる |
| バズる | 結果を保証 | 注目を集める可能性 |
| 集客できる | 結果を保証 | 参加予定を可視化できる |
| 成功する | 結果を保証 | 目標達成の参考になる |
| 確実に | 保証表現 | 〜の目安として |
| 必ず | 保証表現 | 〜の可能性が |
| 保証 | 法的責任 | 参考情報 |

### カテゴリー2: 数値の断定（重要）
| NG表現 | 理由 | OK表現 |
|--------|------|--------|
| 〜になる | 未来確定 | 〜の目安になります |
| 〜できるようになる | 結果約束 | 〜の判断材料になります |
| ◯◯人集まる | 未来確定 | ◯◯人が参加予定です（現時点） |
| ◯◯%達成 | 結果断定 | ◯◯%の参加予定（現時点） |
| この数字なら成功 | 評価断定 | この数字は判断材料の一つです |
| 動員数 | 実績を連想 | 参加予定数 |
| 参加者 | 確定を連想 | 参加予定者 |

### カテゴリー3: 誇張表現
| NG表現 | 理由 | OK表現 |
|--------|------|--------|
| 誰でも | 過度な一般化 | 多くの方が |
| 最短で | 誇張 | 効率的に |
| 効果絶大 | 誇張 | 効果的な |
| 増える（確実に増える） | 結果断定 | 増える可能性がある |
| このアプリで動員できる | 因果断定 | このアプリで参加予定を可視化できる |

## OK表現の置き換えルール

### 価値訴求の言い換え
| 場面 | NG | OK |
|------|----|----|
| トップ/オンボーディング | 「集客を成功させよう」 | 「参加の意思を"見える化"して、判断しやすく」 |
| 価値説明 | 「売れる仕組み」 | 「応援の気持ちを"数"にして整理する」 |
| 効果説明 | 「確実に動員UP」 | 「参加予定の"目安"を把握できる」 |

### ボタン・アクション文言
| 場面 | NG | OK |
|------|----|----|
| 参加ボタン | 「参加する」 | 「参加予定を表明」 |
| 視聴ボタン | 「視聴する」 | 「視聴予定を表明」 |
| 作成完了 | 「成功させる」 | 「盛り上げる」 |

### 数字・統計の表現
| 場面 | NG | OK |
|------|----|----|
| カウンター | 「12人参加」 | 「12人が参加予定（現時点）」 |
| 進捗 | 「50%達成」 | 「50%の参加予定（現時点）」 |
| 地図 | 「東京から5人」 | 「東京から5人が参加予定」 |

## 数字表示の必須ルール

### ルール: 数字のそばに"意味ラベル"を必ず置く

**目的:** 数字だけが独り歩きして誤解・誇張・クレームにつながるのを防ぐ

**実装方法:**
- 数字の直下 or 右に必ず補足を置く
- 「予定」「目安」「現時点」のいずれかを含める

**例:**

✅ **Good**
```
12
参加予定（現時点）
```

```
配信視聴：34
同時視聴予定
```

```
達成率 50%
参加予定ベース
```

❌ **Bad**
```
12人
```

```
配信視聴：34
```

```
達成率 50%
```

## りんく吹き出しのガイドライン

### りんくの人格定義
- **役割**: 状況を整理して伝える案内役
- **NOT**: 決断を促す存在、背中を押しすぎる存在

### りんく口調 NG
- **煽る**: 「絶対行こう！」「今すぐ参加しないと！」
- **背中を押しすぎる**: 「これで成功間違いなし！」
- **成果を断定する**: 「これなら売れるよ！」

### りんく口調 OK
- **翻訳する**: 「これは参加予定の数だよ」
- **整理する**: 「会場参加と配信視聴を選べるよ」
- **安心させる**: 「予定だから、変更もできるよ」

### りんく吹き出しの例

✅ **Good**
```
「応援の気持ちを、予定として"見える化"するよ」
「これは"参加予定"の数だよ。判断の参考にしてね」
「集客を保証するものではないけど、不安を減らす材料になるよ」
「同じ時間を共有する仲間が増えてるね♪」
```

❌ **Bad**
```
「これで絶対売れるよ！」
「今すぐ参加しないと損だよ！」
「この数字なら成功間違いなし！」
```

## 運用チェックリスト

### PR前チェック（5秒）
新しい文言を追加・変更する際に、以下を確認：

- [ ] **成果を断定していないか**
  - 「売れる」「集客できる」「成功する」等
  
- [ ] **数字に「予定／目安」が付いているか**
  - カウンター、進捗バー、ランキング等
  
- [ ] **「できる／成功する」と言っていないか**
  - 未来の結果を約束していないか
  
- [ ] **りんくが煽っていないか**
  - 「絶対」「今すぐ」「損」等の表現

### 実装時の確認ポイント

#### 1. 新しい画面を追加する場合
- [ ] 数字表示に「参加予定（現時点）」等のラベルを追加
- [ ] ボタン文言が「〜予定を表明」形式になっているか
- [ ] りんくの吹き出しが煽っていないか

#### 2. 既存画面を修正する場合
- [ ] 禁止ワードが含まれていないか一括検索
- [ ] 数字のそばに意味ラベルがあるか確認
- [ ] 文言が「判断材料」「可視化」の方向性に沿っているか

#### 3. コピー・告知文を作成する場合
- [ ] 結果を保証していないか
- [ ] 「参加予定」「判断材料」の表現を使っているか
- [ ] 誇張表現（誰でも・最短・効果絶大等）を避けているか

## 一括検索用の正規表現

新しいコードをコミットする前に、以下のパターンで検索：

```bash
# 禁止ワードの検索
grep -rn "売れる\|バズる\|集客できる\|成功する\|確実\|必ず\|保証" app/ features/ components/

# 数値断定の検索
grep -rn "になる\|できるようになる\|集まる\|達成\|この数字なら" app/ features/ components/

# 誇張表現の検索
grep -rn "誰でも\|最短で\|効果絶大\|増える（確実\|動員できる" app/ features/ components/
```
