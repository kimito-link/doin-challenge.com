## テーマ1: Phase 2の次のステップ

### 現状の問題

**二重導線が存在している**:
- トップページに「Xでログイン」ボタン
- ハンバーガーメニューに「Xでログイン」ボタン
- ユーザーがどちらを使うべきか迷う

### Phase 2実装ガイドの思想

> **二重導線が混乱を生む**
> - トップとメニューに「Xでログイン」が2つある
> - どちらを使うべきか分からない
> - **Phase 2で統一する**

（出典: `docs/phase2-implementation-guide.md`）

### 相談内容

1. **トップページのログイン導線をどうすべきか？**
   - 削除してメニューに一本化？
   - トップに残してメニューから削除？
   - 両方残すが、片方を「サブ導線」として扱う？

2. **ログイン導線の統一方針**
   - 「いつでもアクセスできる場所」はどこか？
   - トップページとメニュー、どちらが優先度が高いか？
   - りんくの吹き出しでどう案内すべきか？

3. **Don't Work!思想との整合性**
   - 「増やしたい幸せ」: "押していいボタン"が一目で分かる
   - 「捨てるもの」: "賢そうに見せるための複雑さ"
   - 二重導線は「捨てるべき複雑さ」か？

## テーマ3: カテゴリ/ジャンル設計の最終確認

### 現在の設計

**PURPOSES（目的）**:
```typescript
export const PURPOSES = [
  { id: "live", label: "ライブ・イベント", ... },
] as const;

export const LEGACY_PURPOSES = [
  { id: "streaming", ... },
  { id: "release", ... },
  { id: "birthday", ... },
  { id: "other", ... },
] as const;
```

**設計思想**:
- 新規作成で選べるのは`live`のみ
- 既存データは`LEGACY_PURPOSES`として表示のみ対応
- 将来の拡張は「UI側の解放だけで行える」

（出典: `shared/purposes.ts`、`docs/chatgpt-consultation-category-simplification.md`）

### 相談内容

1. **PURPOSESを`live`のみに絞った設計の妥当性**
   - この設計は「ライブ動員に集中」という思想に合致しているか？
   - ユーザーが「推しの生誕祭」を作成する際に混乱しないか？
   - りんくの吹き出しメッセージは適切か？

2. **GENRESの絞り込み方針**
   - 現在のGENRES: `idol`, `vtuber`, `band`, `voice_actor`, `other`
   - これを「動員に適したジャンル」に絞るべきか？
   - `other`は残すべきか？削除すべきか？

3. **レガシーデータの扱い方**
   - `LEGACY_PURPOSES`の表示方法は適切か？
   - 「カテゴリ未設定」のフォールバック表示は適切か？
   - ユーザーに「レガシーデータ」であることを伝えるべきか？
