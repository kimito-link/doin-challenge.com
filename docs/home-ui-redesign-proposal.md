# ホーム画面UI改善 設計案

**作成日:** 2026年1月21日  
**作成者:** Manus AI  
**対象アプリ:** 動員ちゃれんじ（君斗りんくの生誕祭応援アプリ）

---

## 1. 背景と目的

現在のホーム画面は多機能ですが、情報が多くごちゃごちゃした印象があります。参考にした配信ランキングサイトのように、**シンプルで直感的なUI**に改善し、PC/スマホ両方で使いやすくすることが目的です。

### 1.1 現在の課題

| 課題 | 詳細 |
|------|------|
| 情報過多 | 検索バー、フィルター、カテゴリ、注目チャレンジ、一覧が混在 |
| 優先度の不明確さ | どのチャレンジが「今熱い」のかが一目でわからない |
| 調査機能の配置 | ホームに調査機能があるが、別カテゴリの方が適切 |
| ランキング表示なし | 総合ランキングがホームに表示されていない |

### 1.2 改善目標

1. **TOPに総合ランキングを表示** - 最も盛り上がっているチャレンジを一目で把握
2. **カテゴリ別でアクティブを表示** - タブ切り替えでカテゴリ別のチャレンジを表示
3. **調査機能を別カテゴリに移動** - ホームをシンプルに保つ
4. **PC/スマホ両対応のシンプルなUI** - 操作性を損なわないデザイン

## 2. 参考サイトの分析

配信ランキングサイトの構造を分析し、動員ちゃれんじに適用できる要素を抽出しました。

### 2.1 参考サイトの構造

```
┌─────────────────────────────────────────────────────────┐
│ ヘッダー: サイトタイトル + SNSリンク                      │
├─────────────────────────────────────────────────────────┤
│ ツールバー: 設定ボタン群（ランキング設定、非表示、自動更新等）│
├─────────────────────────────────────────────────────────┤
│ プラットフォームタブ: フルスペ | ABEMA | YouTube | Twitch...│
├─────────────────────────────────────────────────────────┤
│ アイコンフィルター: 各プラットフォームのアイコンボタン      │
├─────────────────────────────────────────────────────────┤
│ ランキングリスト:                                        │
│   1. [サムネ] タイトル / 配信時間 / viewers数            │
│   2. [サムネ] タイトル / 配信時間 / viewers数            │
│   ...                                                   │
└─────────────────────────────────────────────────────────┘
```

### 2.2 参考にすべきポイント

| 要素 | 参考サイト | 動員ちゃれんじへの適用 |
|------|-----------|---------------------|
| タブナビゲーション | プラットフォーム別タブ | カテゴリ別タブ（総合/ソロ/グループ/カテゴリ） |
| ランキング表示 | viewers数で順位付け | 参加者数/進捗率でランキング |
| 視覚的な順位 | 左端に大きな順位番号 | 同様の順位表示を採用 |
| カテゴリバッジ | 「ゲーム」「ブログ」等のタグ | イベントタイプバッジを表示 |
| リアルタイム更新 | 自動更新機能 | 参加者数のリアルタイム更新 |

## 3. 新しいホーム画面の設計

### 3.1 画面構成案

```
┌─────────────────────────────────────────────────────────┐
│ AppHeader: ロゴ + 通知アイコン                           │
├─────────────────────────────────────────────────────────┤
│ メインタブ: 🏆総合 | 👤ソロ | 👥グループ | ⭐お気に入り    │
├─────────────────────────────────────────────────────────┤
│ サブフィルター: カテゴリ選択（横スクロール）              │
├─────────────────────────────────────────────────────────┤
│ 検索バー（折りたたみ可能）                               │
├─────────────────────────────────────────────────────────┤
│ ランキングリスト:                                        │
│   🥇 [サムネ] タイトル / 主催者 / 参加者数 / 進捗バー     │
│   🥈 [サムネ] タイトル / 主催者 / 参加者数 / 進捗バー     │
│   🥉 [サムネ] タイトル / 主催者 / 参加者数 / 進捗バー     │
│   4. [サムネ] タイトル / 主催者 / 参加者数 / 進捗バー     │
│   ...                                                   │
├─────────────────────────────────────────────────────────┤
│ FAB: チャレンジ作成ボタン                                │
└─────────────────────────────────────────────────────────┘
```

### 3.2 タブ構成

| タブ名 | 表示内容 | ソート順 |
|-------|---------|---------|
| 🏆 総合 | 全てのアクティブなチャレンジ | 参加者数 × 進捗率（勢いスコア） |
| 👤 ソロ | ソロアーティストのチャレンジ | 勢いスコア順 |
| 👥 グループ | グループのチャレンジ | 勢いスコア順 |
| ⭐ お気に入り | ユーザーがお気に入り登録したチャレンジ | 登録順 |

### 3.3 勢いスコアの計算式

```
勢いスコア = (参加者数 × 進捗率) + (24時間以内の新規参加者数 × 2)
```

この計算式により、以下のチャレンジが上位に表示されます：
- 参加者が多く、目標達成に近いチャレンジ
- 直近で盛り上がっているチャレンジ

### 3.4 ランキングカードのデザイン

```
┌──────────────────────────────────────────────────────┐
│ 🥇  [サムネ]  タイトル                    👥 123人   │
│              主催者名                     ━━━━━━ 75% │
│              📅 2/14 | 📍 東京           [参加する]  │
└──────────────────────────────────────────────────────┘
```

| 要素 | 説明 |
|------|------|
| 順位アイコン | 1-3位はメダル絵文字、4位以降は数字 |
| サムネイル | チャレンジのサムネイル画像（正方形） |
| タイトル | チャレンジ名（最大2行） |
| 主催者名 | ホスト名 |
| 参加者数 | 現在の参加者数 |
| 進捗バー | 目標達成率を視覚化 |
| 日付・場所 | イベント日時と会場 |
| 参加ボタン | クイック参加ボタン（オプション） |

## 4. 調査機能の移動先

現在ホームにある調査機能は、以下のいずれかに移動することを提案します。

### 4.1 案A: 専用タブを追加

```
タブ: 🏆総合 | 👤ソロ | 👥グループ | ⭐お気に入り | 🔍調査
```

**メリット:** アクセスしやすい  
**デメリット:** タブが増えてスマホで見づらくなる可能性

### 4.2 案B: マイページ内に配置

```
マイページ:
  - プロフィール
  - 参加中のチャレンジ
  - 主催したチャレンジ
  - 🔍 調査・統計  ← ここに追加
```

**メリット:** ホームがシンプルに保たれる  
**デメリット:** 発見しにくくなる可能性

### 4.3 案C: ヘッダーメニューに配置

```
ヘッダー: [ロゴ] .............. [🔍調査] [🔔通知]
```

**メリット:** どの画面からもアクセス可能  
**デメリット:** ヘッダーが複雑になる

**推奨:** 案B（マイページ内に配置）を推奨。調査機能は頻繁に使う機能ではないため、マイページに配置してホームをシンプルに保つのが適切です。

## 5. PC/スマホ対応

### 5.1 スマホ表示（幅 < 768px）

- タブは横スクロール可能
- カードは1列表示
- 検索バーは折りたたみ可能
- FABは右下に固定

### 5.2 PC表示（幅 >= 768px）

- タブは全て表示
- カードは2-3列のグリッド表示
- 検索バーは常に表示
- FABは右下に固定（または上部にボタン追加）

### 5.3 レスポンシブブレークポイント

| ブレークポイント | 列数 | カード幅 |
|----------------|------|---------|
| < 480px | 1列 | 100% |
| 480-768px | 1-2列 | 50-100% |
| 768-1024px | 2列 | 50% |
| 1024-1200px | 3列 | 33% |
| > 1200px | 3-4列 | 25-33% |

## 7. GPTへの相談ポイント

以下の点についてGPTに相談し、設計をブラッシュアップすることを推奨します。

1. **勢いスコアの計算式** - より適切な計算式があるか
2. **タブ構成** - 4タブで十分か、他に必要なタブはあるか
3. **調査機能の配置** - 案A/B/Cのどれが最適か
4. **ランキングカードのデザイン** - 情報の優先順位は適切か
5. **PC/スマホの切り替え** - ブレークポイントは適切か

---

## 8. まとめ

本設計案では、配信ランキングサイトを参考に、以下の改善を提案しました。

1. **総合ランキングをTOPに表示** - 勢いスコアでソートしたランキング形式
2. **タブでカテゴリ切り替え** - 総合/ソロ/グループ/お気に入りの4タブ
3. **調査機能をマイページに移動** - ホームをシンプルに
4. **PC/スマホ両対応** - レスポンシブデザインで操作性を確保

この設計案をベースにGPTと相談し、より良いUIを目指します。
