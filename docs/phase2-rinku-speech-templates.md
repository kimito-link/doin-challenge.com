# りんく吹き出しテンプレート集

## 使い方

このテンプレート集は、アプリ全体でりんくキャラクターの吹き出しを統一するためのものです。

### 基本ルール
1. **絵文字は使わない**
2. **親しみやすいトーン**（「だよ・だね」調）
3. **ポジティブな表現**
4. **専門用語は避ける**
5. **短く、わかりやすく**

## 文言作成のガイドライン

### トーンの統一

#### ✅ 良い例
- 「できたよ！」
- 「試してみてね！」
- 「ありがとう！」
- 「またね！」

#### ❌ 悪い例
- 「完了しました。」（堅すぎる）
- 「エラーが発生しました。」（専門用語）
- 「もう一度お試しください。」（丁寧すぎる）
- 「😊」（絵文字禁止）

### 文の長さ

#### 理想的な長さ
- **1文**: 15〜30文字
- **2文**: 合計40〜60文字
- **3文以上**: 避ける（長すぎる）

#### ✅ 良い例
```
ログインすると、参加履歴やお気に入りが見られるよ！
```
（28文字）

#### ❌ 悪い例
```
ログインすると、参加したチャレンジの履歴を確認したり、お気に入りに追加したチャレンジをいつでも見られるようになります。
```
（60文字、長すぎる）

### 改行のルール

#### 基本ルール
- 1文ごとに改行
- 2文以上の場合は空行を入れない

#### ✅ 良い例
```
ログアウトすると、参加履歴やお気に入りが見られなくなるよ。
本当にログアウトする？
```

#### ❌ 悪い例
```
ログアウトすると、

参加履歴やお気に入りが見られなくなるよ。

本当にログアウトする？
```
（空行が多すぎる）

## 実装時の注意事項

### コンポーネントの使用

```tsx
import { LinkSpeech } from '@/components/organisms/link-speech';

// 基本的な使い方
<LinkSpeech>
  ログインすると、参加履歴やお気に入りが見られるよ！
</LinkSpeech>

// 複数行
<LinkSpeech>
  ログアウトすると、参加履歴やお気に入りが見られなくなるよ。{'\n'}
  本当にログアウトする？
</LinkSpeech>
```

### 文言の管理

1. **このテンプレート集を参照**
   - 新しい画面を作る時は、まずこのテンプレート集を確認
   - 似た状況の文言を再利用

2. **新しい文言を追加する場合**
   - このドキュメントに追加
   - トーンとルールを守る
   - レビューを受ける

3. **文言を変更する場合**
   - このドキュメントを更新
   - 影響範囲を確認
   - 一貫性を保つ

## まとめ

このテンプレート集は、りんくキャラクターの「声」を統一するためのものです。

### 守るべき3つのポイント
1. **絵文字は使わない**
2. **親しみやすく、でも丁寧すぎない**
3. **短く、わかりやすく**

新しい文言を作る時は、このテンプレート集を参考にしてください。
